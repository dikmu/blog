Oracle SOA 12c加速面向服务架构实现








面向设计服务架构可以用于创建灵活的、可管理的企业系统，但是技术及消费化的新趋势已经改变了SOA系统的需求。如果使用面向服务架构的组织希望维持当前继续满足用户和利益相关者需求的系统，那么面向SOA的架构必须满足移动和社交媒体方面的要求，与此同时也能够适应和融合云技术。

然而面对云时代，移动互联，物联网的时代，企业与架构师在对待SOA时往往有存在一些误解，由于这些误解的存在，企业内面相服务的架构与云，移动，物联网结合的时候，一些常见的绊脚石往往被过度扩大了。一旦理清了SOA的这些误解，再借助于业内主流提供商的SOA产品，就能发现，加速SOA的实现，能极大加速企业IT向移动互联网、物联网和云的过度。

误解一：SOA已趋于死亡，API、微服务才是未来

事实上，无论是微服务论还是REST论的争议由来已久，但主流的声音还是认为无论是微服务，还是REST的出现，并不是用来取代SOA的。SOA将转向支持物连、移动化和云。人们将用它支持面向特定任务的服务，它们称为应用程序接口-APIs。这些API用于移动设备和云计算解决方案。一个典型例子是web页面上的“喜欢”或“发到朋友圈”按钮。这些系统通过API进行交互，现在，API常常作为SOA的“门面”，由SOA提供整合、安全性、治理和管理等服务。

误解二：SOA很复杂，不好用

这绝对是个谬论，SOA并不复杂。当SOA出现还是少数学院派口中的名次时，虽然思想上各家都能达成共识，但是各自的实现却不一致。加之一些产品仅仅是把多个功能部件简单整合，光是SOA产品本身的使用和管理就有多种组合和多种工具，这一阴影在相当长一段时间内是架构师和开发者挥之不去，甚至时至今日，还在影响他们的选择。事实上，成熟的SOA产品会提供简单有效的工具，通过服务的编排，整形、路由，策略等实现复杂应用和业务流程的组合，以提供更好的敏捷性。甲骨文SOA提供了单一的安装包和全可视化开发工具JDeveloper以及DevOps 的支持，30分钟即可实现从零开始的环境部署，这已经达到和接近于PaaS和SaaS的部署复杂度。额外的，甲骨文SOA云服务也可实现传统SOA服务向云端的一键切换。这么一说，SOA远没想象的复杂，不是吗？问题的关键是要选对产品和方案。

误解三：SOA仅仅一项技术

SOA对于企业IT来说，这意味着一种技术架构，意味着服务开发设计的思想。事实上SOA的软件提供商正是从设计架构角度，借助于成熟工具，封装并屏蔽晦涩的技术。通常现代化的程序技术发展如移动化、物联网等等都会带来很多新的技术。比如大数据听起来似乎和SOA没啥关系，但是低延迟、实时通讯、复杂事件处理、事务处理等都已经在 SOA中实现，通过分析并针对事件运用规则的方式解决大数据问题。所以，无论是当前前沿科技还是传统技术，甲骨文SOA中“包容兼蓄，共生共荣”的特性能够稳定为企业输出灵活IT架构，制成未来业务发展转型。SOA中的服务梳理和分析，服务流程优化，服务管控和治理都将成为有效支撑企业服务架构。相比无论是微服务，还是REST，SOA侧重点远不是单一的技术层面，而是创建以新技术创新为驱动，稳健基础平台为支撑的全新的未来企业IT架构。

既然SOA是设计原则，那么它又饱含哪些内容与更新呢。事实上，甲骨文公司一直致力于在模糊性和宽泛性的SOA上总结原则与方法，归纳并标准化。在如今客户机－服务端模式逐渐向互联网、云模式变革演进过程中，认清方向，抓住本质。在掌握和了解宏观SOA概念同时，我们更要着眼于SOA落地，那我们我们来看看甲骨文SOA套件如何更加贴近实际、贴近用户。

在过去一年里，Oracle已经正式发布了SOA Suite 12c(12.2.1)，带来了新的组件功能更新与增强。大部分新增的功能都具有重大意义且非常实用。Oracle SOA 12c的发布将从四大方面为潜在观望用户和既有用户带来全新的使用体验：

从产品和用户使用：

1. 更具生产力的开发环境 与集成

2. 更高性能、更稳定

从功能趋势角度：

3. 移动化、IoT 集成

4. 云集成


图1 SOA Suite 12c全面引入既有系统与Cloud、Mobile，IoT的集成

下文将带大家一起快速熟悉下Oracle SOA Suite 12c在以上几方面所带来的激动人心的新特性以及其它关键更新

（一）生产力全面提升的开发环境与集成

  • 简化的一键安装部署QuickStart

区别与传统的SOA开发环境需要有安装和部署来实现SOA的开发环境不同，Oracle SOA 12c为开发人员提供单一的安装包，包含

  • JDeveloper

  • Weblogic

  • SOASuite（含 Service Bus）

  • JavaDB

  • EM FMWControl

用户仅仅需要简单回答几个问题，如安装路径，OracleHome变量等，即可在数十分钟内完成开发环境的部署


图2 SOA Quick Start 安装

  • 统一的可视化申明式开发工具集JDeveloper

Oracle SOA 12c继续提供统一的用户界面可视化开发工具用于SOA，Service Bus和Event Processing等关键SOA组件设计与开发，该特性大大简化和消除了培训成本，为更快速实现整合提供了可能。

  • OracleService Bus可视化申明式开发

Oracle SOA 12c中组件如Service Bus，OEP，BPEL等全部拥有可视化的开发工具，采用申明式＋向导的方式完成定义和参数设置


图3 Service Bus在JDeveloper中的视图

  • 模版和BPEL subprocesses

“分享”和”重用”是SOA架构的一大核心思想。Oracle SOA Suite 12c引入了模板的功能来加强分享和重用的功能。

  • 调试器与测试框架的增强

卓有成效的开发流程需要有简单的调试与测试工具。JDeveloper包含了一个可视化的调试工具可以在composite，BPEL process或者是pipeline中打断点调试，图x显示了一个BPEL Process包含了一个SOAP接口和2对断点


图4 SOA构件中的断点（红色箭头标示）

与JavaIDE中的调试器相同，SOA调试器提供了诸如stepping into，out，over等，同时提供了变量查看器，也能实时修改变量值


图5调试模式下的变量视图

SOA Suite test framework在之前的版本中已经发布，在SOA 12c中得到了增强，输入和输出能够自动生成或者从样本文件中自动加载；能够对请求和响应还有异常进行断言，能够模拟外部服务和异常。测试能够在JDeveloper中被直接执行和直接显示每次执行的详细的结果，无需在开发工具和监控平台中反复切换查看与分析。


图6 JDeveloper中的测试结果视图

（二）平台更加稳定，性能增强

SOA构件延迟加载，Oracle SOA 12c引入了一个全新的特性称之为“构件延迟加载“。这是一个很棒的特性，允许资源文件或组件在第一次使用时才被真正加载，这能够极大提升服务器的启动速度特别是当SOA域中部署了大量服务构件的时候。

在Weblogic中大受欢迎的工作管理器也接管了SOA 12c中绝大部分的相关工作线程，利用工作管理器的自调优和精准配置

（三）移动化集成，服务移动化

企业移动化的需求随着移动终端设备的普及而日益显著，对于企业应用的使用，从传统的PC浏览器转变为随时、随地的交互要求。为了应对日益迫切的企业应用移动化挑战，REST/JSON已经成为移动服务和API提供的事实标准。Oracle SOA 12c引入了REST绑定，通过JDeveloper，能够非常简便的实现传统SOAP服务，EJBs，JCA适配器连接的传统应用启用移动化支持，即自动转化为RESTful接口与服务。


图7 Oracle Service Bus把一个SOAP后台服务导出为一个REST服务

Oracle SOA 12c中，Oracle Event Processing（OEP）被集成到JDeveloper，通过图形化的面板，使用拉拽的方式使用帮助向导即可完成定义和参数的设置。另一个关键功能Event Delivery Network（EDN）事件适配器也被加入，可以非常简便的完成与SOA构件及其它SOA组件的集成。


图8 可视化EPN设计开发环境

（四）云端集成

快速实现不断增长的云应用和云服务与企业内部应用和数据的集成

  • Oracle Cloud Adapter

Oracle SOA 12c致力于简化、加速并优化与各种云端应用的集成，不仅会提供基于标准化的集成平台用于连接，更提供了强大的一整套解决方案用于解决审计、合规、安全和治理等。Oracle SOA 12c内建了基于标准化连接 云端应用和企业内部传统应用以及其它云端应用。连接器隐藏了用于连接、安全、和会话管理的额外编码和重复配置，同样也避免了与云应用交互时所需的额外复杂技术陷阱与障碍。所以这些连接器能够解决面对所有用于云端应用集成中所需要面对的问题，开发者仅仅只需要关注连接的逻辑和业务流程。


图9 Salesforce云适配器配置截图

  • Cloud Adapter SDK

除了提供云应用适配器外，Oracle SOA 12c还包含了Cloud Adapter SDK。SDK能够帮助客户及其合作伙伴快速融入新的SaaS服务到企业业务逻辑中。目前已经提供的诸如Salesforce，Eloqua，RightNow ，OracleSales Cloud，NetSuite，Oracle ERPCloud等等也都基于此SDK开发。


图10为云应用集成定制的SDK，简化并加速云服务集成

总结：

Oracle SOA suite12c提供了业界最高集成能力的平台用于加速实现企业基于云、移动化和物联网的面向服务架构平台。在全面拥抱新技术新平台的同时，Oracle SOA 12c致力于最大限度地保留且扩展现有的技术架构体系、技能和工具套件。通过使用Oracle SOA 12c，无论您是最终用户还是合作伙伴，在面对移动化，物联网设备的爆炸式增长的挑战时，能够持续稳定为企业关键业务提供更快，更低成本的解决方案，并大大降低整体的拥有维护成本。
